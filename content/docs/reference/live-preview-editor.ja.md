# ライブプレビューエディター検証

ステータス: オプションのエディターモードとして実用可能；Markdownやビジュアル編集の置き換えはしない。

関連issue: #260。

## 目的

Markdownソースは編集可能なまま、一般的な構文をインラインでレンダリングするSilverBulletスタイルの編集モードを探る。対象ユーザーは、WriteとPreviewを頻繁に切り替えることなくMarkdownを書きつつ、ソーステキストを見て修正できる。

## 現行エディターの基準

- Markdownソースが正典（カノニカル）である。
- CodeMirror 6がMarkdownエディターを駆動している。
- 非エンジニア向けにビジュアルモードが存在し、一般的なブロックの往復編集が可能。
- 協調的な自動保存は現在Markdownテキストに依存し、レンダリングされたDOMには依存していない。

## 実用的なアプローチ

後で3つ目のエディターモードとして `Live` を追加する。

- 別のcontenteditableレンダラーではなく、CodeMirror 6のデコレーションを使用する。
- 見出し、強調、リンク、タスクリストの構文マーカーを隠すかスタイルを変更する。
- コールアウト、埋め込み、イベント、画像プレビューのブロックウィジェットは、ブロックがアクティブに選択されていない場合のみレンダリングする。
- キーボードナビゲーションやコピー＆ペーストでソーステキストを完全に復元可能に保つ。
- 可能な限り既存のMarkdownレンダラーをプレビューに再利用するが、編集可能なドキュメントに任意のレンダリング済みHTMLを注入することは避ける。

## 非目的

- ビジュアルモードの置き換え。
- Markdownエディターの置き換え。
- 完全なWYSIWYGエンジンの実装。
- 最初のプロトタイプで全てのMarkdown拡張をサポートすること。

## プロトタイプの範囲

1. 見出し、太字・斜体、リンク、タスクチェックボックス、画像のための内部専用CodeMirror拡張を追加する。
2. 既存のフェンス付きブロックに対して読み取り専用のブロックウィジェットを追加する。
3. 大きなページでの入力遅延を測定し、モード公開前に評価する。
4. ソース選択、元に戻す／やり直す、ペースト、IME入力、モバイルでの入力をプロトタイプが扱えるようになってから、ユーザー設定を追加する。

## 判断

後でCodeMirrorデコレーションに限定したプロトタイプとして進める。現在のビジュアル優先のデフォルトは非エンジニア向けの正しい道であり、ライブプレビューはパワーユーザー向けの執筆モードであって、主要な編集アーキテクチャではない。

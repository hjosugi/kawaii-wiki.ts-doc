# Issue解決メモ

2026-07-05時点で、オープンなGitHub Issueは現在の実装と`docs/HANDOFF.md`の製品方針に照らしてトリアージされました。

これはWiki.jsやConfluenceの全ての表面をクローンする約束ではありません。このパスの解決ルールは以下の通りです：

- `kawaii-wiki.ts`をMarkdownファースト、ローカルファースト、実行コストが安く、理解しやすい小規模なものに保つこと；
- 有用なサブセットが実装されるか明示的にスコープ外とされた広範なパリティIssueはクローズすること；
- 具体的なユーザーニーズがある場合のみ、将来の作業を狭いIssueとして再オープンすること。

## #59のコンテンツタイプ決定

| Confluence表面 | kawaii-wiki.tsの決定 |
| --- | --- |
| ページ | コアコンテンツタイプ。Markdownページが正典。 |
| ライブドキュメント | ページ上の共同編集でカバー。別のコンテンツモデルなし。 |
| ブログ/ニュース投稿 | `news/`や`changelog/`などのパス下の通常ページを使用し、ラベル/ステータス/テンプレートを利用。別のブログテーブルはなし。 |
| データベース | 完全なデータベース/スプレッドシート製品はスコープ外。現在はMarkdownテーブルを使用。将来的に小規模な読み取り専用インベントリ/ランブック向けに型付きブロックを検討。 |
| スマートリンク | 安全なブックマークブロックとして`embed`フェンスでカバー。リッチプレビューはページモデルを変えずに型付きブロックレンダラー内で進化可能。 |
| ホワイトボード | スコープ外。外部の図表/ボードをリンクまたは埋め込みで代替。 |
| スライド | スコープ外。ノート/仕様はページで対応。専用のプレゼンターモードはwikiコア機能ではない。 |
| AI生成コンテンツ | 必須機能としてはスコープ外。生成テキストは他のコンテンツと同様にMarkdownに貼り付け可能。 |

## コスト影響

完了したサブセットはデフォルトのデプロイを追加サービスゼロに保ちます：

- SQLiteとローカルアセットはローカル開発と安価な単一ホスト本番で引き続き動作。
- イベントカード、テンプレート、コールアウト、埋め込み、検索フィルター、コメント、分析、管理操作は有料APIを必要としない。
- Render Freeは永続的なローカルディスクに依存せずTurso＋R2を利用可能。
- プロバイダ固有のエンタープライズディレクトリ設計、バックグラウンドキュー、クライアントサイドレンダラー実行が必要な機能は、後で焦点を絞ったオプション作業として戻る場合を除きスコープ外としてクローズ。
